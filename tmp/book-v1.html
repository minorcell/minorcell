simple_chatbot.py
import re
def eliza_respond(text):
# 最早的聊天机器人ELIZA的核心逻辑
# 只有约200行模式匹配规则
text = text.lower()
patterns = [
(r'我(.*)难过',
'为什么你觉得难过？'),
(r'我(.*)开心',
'是什么让你感到开心？'),
(r'(.*)你(.*)懂(.*)',
'你觉得我懂吗？'),
(r'(.*)妈妈(.*)',
'说说你的家庭吧'),
(r'我(.*)觉得(.*)',
'你为什么觉得\g<2>？'),
(r'因为(.*)',
'这个理由充分吗？'),
]
for pattern, response in patterns:
m = re.match(pattern, text)
if m:
return response
return '请继续说下去...'

卷一 · 机器梦见智能 · 1940s—1980s

第 1 章：图灵的问题：机器会思考吗？

本章问题：机器能不能表现得像人在思考？

1950 年，艾伦·图灵提出了一个看起来简单、实际上很难回答的问题：机器会思考吗？这个问题麻烦在于，每个词都不安分。什么是"机器"？是一台会转动齿轮的机械装置？是一台执行程序的电子计算机？还是任何能够处理信息的系统？

什么是"思考"？是做数学题？是理解语言？是拥有意识？是能解释自己的理由？还是只要表现得足够聪明就算？

如果我们一开始就纠缠这些定义，讨论很快会变成哲学泥潭。每个人都可以提出自己的定义，然后谁也说服不了谁。

图灵聪明的地方在于：他没有硬着头皮给"思考"下定义。他换了一个问题。

模仿游戏

图灵把他的测试称为"模仿游戏"（The Imitation Game）。游戏的设定大致是这样的：一个提问者通过文字终端和两个对象交流——其中一个是人类，另一个是机器。提问者可以向双方提任何问题，但只能通过文字。

如果经过一段时间的对话，提问者无法可靠地分辨谁是人类、谁是机器，那么这台机器就通过了测试。

请注意图灵没有说的东西：他没有说通过测试就意味着机器"真的"有意识。他没有对"智能"下任何形而上的定义。他只是把一个哲学困境变成了一个工程问题——这让后来的人工智能研究有了可操作的目标。

最早的"聊天机器人"：ELIZA

1966 年，MIT 的 Joseph Weizenbaum 写了一个程序叫 ELIZA。它模拟一个心理医生，用非常简单的模式匹配规则来回应人类输入。它只有大约 200 行代码。

然而，让 Weizenbaum 震惊的是：很多人真的把 ELIZA 当成了人类。他的秘书甚至请他离开房间，因为她觉得和 ELIZA 的对话是"私密的"。Weizenbaum 后来成为了 AI 的尖锐批评者。

ELIZA 揭示了一个深刻的事实：人类非常容易将智能投射到任何表现出类似对话行为的系统上。这个现象后来被称为"ELIZA 效应"——它至今仍然是大模型交互中最重要的心理学事实之一。

动手试试

现在，你可以亲自体验一下和 ELIZA 对话的感觉。左侧面板已经变成了一个聊天窗口——背后的逻辑就是上面那些简单的模式匹配规则。

试试说"我很难过"、"我觉得不开心"、"因为工作太累"……

当你真的和它对话之后，再回想 1966 年的情景：那些从未接触过计算机的普通人，面对这样一个程序，真的相信对面坐着一个关心他们的心理医生。

从图灵到 ChatGPT

图灵测试是 1950 年提出的。ELIZA 是 1966 年的。ChatGPT 是 2022 年的。

七十二年。从"机器会思考吗"到"ChatGPT 真的理解吗"，人类问的是同一个问题。但在这个过程中，我们积累了几代人的努力：感知机、反向传播、卷积网络、词向量、Transformer、RLHF……
