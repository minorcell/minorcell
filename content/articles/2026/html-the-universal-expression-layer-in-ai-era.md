## 观察

最近我注意到一个有趣的趋势：越来越多的场景开始用 HTML 来承载内容，而这些场景以前跟"网页"毫无关系。

**第一个例子是做PPT。** 以前我们在 Office 或 WPS 里面编辑幻灯片，费时费力。现在出现了一种新的工作流：让 AI 帮你梳理要表达的内容，生成一份 HTML 文件，最后再用工具导出为 PPT。整个过程快得多，而且修改起来非常方便。

**第二个例子是做海报。** 以前一张海报，设计师可能要花一两天，反复返工。后来 AI 能直接生成海报图片了，但问题是改起来很麻烦——你说"把标题往左移一点"，它可能把整张图都改变了。但如果海报是用 HTML 写的，标题就是一个标签，位置就是一行 CSS，AI 可以精确地只改你想改的地方。

**第三个例子是做视频。** B站和 YouTube 上很多知识区 UP 主，视频里有大量的图文内容。以前他们用 PPT 边播放边讲，现在可以先用 HTML 做出图文页面，再导出为视频素材。

**第四个例子是写文档。** 最近 Claude Code 的工程师提到，AI 写的设计文档动辄好几百行，超过两百行人就看不下去了。但如果把同样的内容做成一个 HTML 页面，加上折叠、导航、视觉层次，阅读体验会好很多。

这些例子放在一起，指向一个结论：**HTML 的用途正在远远超出"写网页"的范畴。**

## 为什么是 HTML

在所有的文件格式里，为什么偏偏是 HTML 被 AI 时代重新激活了？

我想了想，觉得答案很朴素。HTML 可能是人类发明过的最好的"结构化表达"格式。它有几个独特的性质：

> 1. **纯文本**——AI 天然就能生成文本。
> 2. **结构化**——有语义、有层次、有嵌套。
> 3. **所见即所得**——浏览器就是渲染引擎，零成本预览。
> 4. **内容与样式分离**——改内容不影响样式，改样式不影响内容。
> 5. **生态成熟**——全世界每台电脑都有浏览器。

对比一下其他格式：PPT 的 `.pptx` 本质上是一堆压缩的 XML，格式极其复杂；Photoshop 的 `.psd` 是二进制的；视频就更不用说了。这些格式有一个共同特点：**它们是为软件设计的，不是为人设计的。** 人类读不懂，AI 生成起来也很吃力。

HTML 不一样。它是 AI 和人类之间摩擦最小的视觉表达协议。

## 范式转移

仔细想想，上面那些场景本质上是同一件事。它们共同指向一种正在出现的新工作流：

**传统路径：** 人脑构思 → 人手操作专业软件 → 输出专有格式

**新路径：** 人脑构思 → AI 生成 HTML → 工具转换为目标格式

当然，现在用新路径的还是少数，大多数人仍然在用传统方式。但这条路径之所以值得关注，是因为中间那一层的替换逻辑非常清晰：从"人操作专业软件"变成了"AI 写 HTML"。

这个转变之所以成立，是因为 AI 的能力特点和 HTML 的格式特点恰好互补：

| AI 擅长的 | HTML 提供的 |
|---------|----------|
| 生成文本 | 纯文本格式 |
| 理解语义结构 | 语义化标签 |
| 精确修改局部 | DOM 树的局部可编辑性 |
| 一次性生成完整作品 | 自包含的单文件渲染 |

海报那个例子最能说明问题。AI 直接生成图片，本质上是把所有信息烧死在像素里了。你没法跟 AI 说"把标题往左移 20 像素"，因为它不知道哪些像素是标题。但 HTML 里，标题就是 `<h1>`，位置就是 `left: 20px`，一切都是可寻址的、可修改的。

**HTML 让 AI 的输出从黑箱变成了白箱。**

## 文档的例子最值得深思

Claude 工程师那个例子，我觉得特别有启发。

一份设计文档，用 Markdown 写，超过 200 行人就不想看了。但做成 HTML 网页，加上折叠、导航、高亮、交互，同样的信息量，认知负担大幅下降。

这说明一个重要的事情：**Markdown 是为阅读设计的，HTML 是为体验设计的。**

当信息复杂到一定程度，纯线性的文本就不够了，你需要折叠和展开来控制信息密度，需要导航和锚点来做非线性跳转，需要颜色和排版来建立视觉层次，甚至需要简单的交互，比如 tab 切换和搜索过滤。

这些事情 Markdown 做不到，Word 做起来很笨重，但 HTML 天生就是干这个的。

而且注意，这里的 HTML 不需要后端，不需要数据库，不需要部署。一个 `.html` 文件，双击就能在浏览器里打开。分发成本几乎为零。

## 更大的图景

如果把视野再拉远一点，我觉得正在发生的事情可以这样概括：

> **HTML 正在从"Web 的展示层"进化为"AI 时代的通用表达层"。**

过去 30 年，HTML 被锁定在"网页"这个心智模型里。我们一想到 HTML，就想到网站、前端、浏览器。但 AI 打破了这个锁定。

AI 不在乎格式叫什么名字，它只在乎一个问题：什么格式我能写、人类能看、还能方便地改？答案指向 HTML。

于是 HTML 开始侵入一切领域：

- **文档** → 取代部分 Markdown 和 Word
- **演示** → 取代部分 PPT
- **设计** → 取代部分 Figma 和 Photoshop
- **视频脚本** → 作为视频的中间表示层
- **数据可视化** → 取代部分 BI 工具
- **原型设计** → 取代部分设计稿

这不是 HTML 变强了。**是 AI 让 HTML 的生产成本趋近于零了。**

过去 HTML 的问题从来不是能力不够——CSS 能画出极其精美的界面，JavaScript 能实现极其复杂的交互——问题是写起来太麻烦了。一个好看的海报页面，手写 HTML/CSS 可能要半天。但现在 AI 十秒钟就能生成。

当生产成本归零，表达能力就被彻底释放了。

## 一个历史回响

最后说一个有意思的事情。

Tim Berners-Lee 在 1989 年发明 HTML 的时候，他的初衷并不是"做网站"。他在 CERN（欧洲核子研究中心）工作，他的初衷是让物理学家们能够方便地共享和展示研究信息。

三十多年过去了，HTML 兜了一大圈，经历了静态网页、动态网页、Web App、单页应用、前端工程化……最后在 AI 时代，反而回到了它最初的使命：

**一种让信息变得可见、可读、可交互的通用格式。**

只不过这次，写它的不再只是人类，还有 AI。
